# Dashboard Design

版本：v0.5

## 目標
Dashboard 作為系統入口，提供「跨模組即時概況」與「可行動告警」：
1. 專案狀態
2. 測試結果
3. 部署狀態
4. AI 修復任務
5. 系統告警

## 後台 Dashboard（`/admin/dashboard`）
### 版面配置
1. Top KPI：專案數、進行中任務數、測試通過率、部署成功率、今日告警數
2. Middle Left：專案進度與里程碑達成率
3. Middle Right：測試趨勢（Unit/E2E）與失敗原因分佈
4. Bottom Left：部署時間軸（最新版本、灰度進度、回滾事件）
5. Bottom Right：AI 修復任務佇列與處理時長
6. Side Panel：高優先級告警（可立即跳轉處理）

### 可互動項
1. 時間範圍篩選：今日/7日/30日/自訂
2. 專案篩選：全部或單一專案
3. 點擊卡片可 drill-down 到對應列表頁
4. 告警支援「指派處理人」「標記已處理」

## 前台 Dashboard（`/`）
### 版面配置
1. 我的任務（待處理、逾期、今日截止）
2. 近期參與專案進度
3. 最新測試與部署通知
4. 我的 AI 互動紀錄摘要

### 可互動項
1. 快速新增任務
2. 一鍵進入 AI Workbench
3. 任務卡片支援快速更新狀態

## 元件設計規範
1. 指標卡片：顯示數值、趨勢箭頭、同比百分比
2. 圖表：所有圖表可匯出 PNG/CSV
3. 告警清單：至少含等級、來源模組、發生時間、處理狀態

## 刷新策略
1. KPI 與告警：30 秒輪詢
2. 圖表：120 秒輪詢
3. 重大事件（發版成功、回滾、測試全紅）即時推播
