# Form Design Guidelines

版本：v0.5

## 基本原則
1. 表單需包含輸入驗證、欄位說明、錯誤提示
2. 支援草稿保存（長表單）
3. 送出前顯示摘要確認（高風險操作）

## 版面規範
1. 欄位分組：基本資訊、進階設定、權限/通知設定
2. 單欄優先，必要時雙欄（桌面）；行動端固定單欄
3. 主要按鈕置右（儲存/送出），次要按鈕置左（取消/返回）

## 欄位規範
1. 必填欄位顯示 `*`，並在送出時做完整驗證
2. 每個欄位提供 placeholder 或 help text
3. 可列舉欄位（角色、環境、狀態）使用下拉選單，不允許自由輸入
4. 長文本欄位（描述、Prompt）提供字數計數器

## 驗證規則
1. 即時驗證：格式錯誤立即提示
2. 提交驗證：阻擋資料不完整的提交
3. 伺服端驗證：錯誤訊息需映射到對應欄位
4. 唯一值（專案代碼、帳號、角色代碼）需檢查重複

## 錯誤與回饋
1. 欄位錯誤：欄位下方顯示紅字錯誤訊息
2. 全域錯誤：頂部 Alert 顯示錯誤摘要與錯誤碼
3. 成功回饋：Toast + 操作結果（例如「已建立任務 #TASK-1024」）

## 重要操作保護
以下操作需要二次確認 Dialog：
1. 停用帳號
2. 刪除專案/任務
3. 發版與回滾
4. 變更系統參數
5. 刪除或輪替 API Key

## 可用性與無障礙
1. Tab 順序符合視覺閱讀順序
2. 所有控制項提供鍵盤操作
3. 錯誤提示需可被螢幕閱讀器讀取

## 安全規範
1. 密碼與金鑰欄位預設遮罩
2. API Key 僅首次顯示完整值，後續只顯示片段
3. 防止重複提交：提交後按鈕進入 loading 並禁用

## 表單範本（建議）
1. 建立類：Project Create、Task Create、User Create
2. 編輯類：Project Edit、Role Edit、Settings Edit
3. 執行類：Deploy Release、Rollback、Restore Backup
