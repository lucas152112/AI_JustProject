# Frontend Pages

版本：v0.5.1

## 設計目標
前台以個人工作台為核心，將任務、專案、AI 協作、測試與通知整合成低切換成本的日常操作介面。

## 前台頁面清單（完整）
| 頁面 | 路由 | 目標角色 | 主要區塊 | 核心操作 |
|---|---|---|---|---|
| 首頁 | `/` | 全部登入使用者 | 我的待辦、專案捷徑、近期通知、個人指標 | 進入任務、快速回報進度 |
| 專案列表 | `/projects` | PM/Dev/Tester/DevOps | 搜尋篩選、專案卡片/表格 | 進入專案、收藏專案 |
| 專案詳情 | `/projects/:projectId` | 專案成員 | 專案概況、里程碑、成員、活動流 | 留言、更新里程碑狀態 |
| 任務列表 | `/tasks` | 全部專案成員 | 我的任務、追蹤任務、篩選器 | 建立任務、認領任務、更新狀態 |
| 任務看板 | `/tasks/board` | PM/Dev/Tester | Kanban 欄位、泳道、WIP 提示 | 拖拉卡片、快速編輯 |
| 任務詳情 | `/tasks/:taskId` | 任務相關人員 | 描述、子任務、評論、Commit 關聯、測試摘要 | 留言、變更狀態、上傳附件 |
| AI 互動頁 | `/ai-workbench` | PM/Dev/Tester | Prompt 編輯器、回應結果、歷史紀錄、模型選擇 | 發送提示、保存模板、建立修復任務 |
| 測試結果 | `/testing` | Dev/Tester/PM | Unit/E2E 結果、失敗案例、趨勢圖 | 重跑測試、建立缺陷 |
| 部署觀察 | `/deployments` | DevOps/PM/Operator | 版本狀態、環境健康度、部署時間軸 | 查看版本差異、訂閱部署通知 |
| 進度報告 | `/reports/my` | 全部專案成員 | 我的進度、專案進度、週報摘要 | 匯出報告、推送至通知通道 |
| 通知中心 | `/notifications` | 全部登入使用者 | 通知列表、篩選、優先級標記 | 標記已讀、批次清除 |
| 個人資料 | `/profile` | 全部登入使用者 | 基本資料、密碼設定、MFA 設定 | 更新資料、更新安全設定 |

## 頁面模式
1. Workspace 型：首頁、任務列表、通知中心
2. Detail 型：專案詳情、任務詳情、個人資料
3. Tool 型：AI Workbench、測試結果、部署觀察

## 前台資訊架構原則
1. 優先呈現「我的任務 / 我的通知 / 我的指標」
2. 跨頁快速操作統一放在右上角
3. 列表與詳情可快速切換，避免深層返回
4. 高頻區塊需支援桌面雙欄與行動端單欄重排

## 響應式規範
1. `>=1280px`：側欄 + 內容雙欄
2. `768px-1279px`：收合側欄，保留 sticky filter
3. `<768px`：單欄卡片流，主要操作改底部浮動按鈕

## 關鍵頁面設計要點
### 首頁
1. 頂部顯示個人 KPI 與今日待辦
2. 中段以專案捷徑與通知並列
3. 底部顯示最近測試/部署動態

### 任務詳情
1. 左側為描述、附件、Checklist
2. 右側為狀態、指派、Git 關聯、測試摘要
3. 底部以時間序呈現評論與活動流

### AI Workbench
1. 左欄為 prompt 模板與歷史
2. 主區為輸入/輸出編輯器
3. 右欄為 token、成本、採納狀態

## 前台操作流程（核心）
1. 使用者進入首頁 -> 查看待辦 -> 進入任務詳情 -> 更新狀態
2. 使用者在任務詳情啟動 AI 互動 -> 產生建議 -> 回寫任務評論
3. 使用者查看測試結果或部署狀態 -> 追蹤風險 -> 回報進度

## 設計一致性
1. 主行動按鈕固定使用頁面右上或底部浮動區
2. 所有列表支援搜尋、篩選、排序、儲存視圖
3. 狀態色與圖示需與後台一致
