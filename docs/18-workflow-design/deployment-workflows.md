# Deployment Workflows

版本：v0.5.1

## 流程目標
定義從測試通過、建置映像、部署到環境升級與回滾的標準交付流程。

## 主要角色
- CI/CD System：執行 pipeline、建置與發版
- Developer：處理部署失敗或回滾需求
- QA：確認測試環境與預備驗收
- Ops / SRE：監控環境狀態與變更風險

## 環境階段
1. `dev`
2. `test`
3. `pp`
4. `prod`

## 主流程
1. 程式碼合併後觸發 Tekton pipeline
2. 建置 artifact 與 container image
3. 執行必要測試、掃描與版本標記
4. 部署到 `dev` 驗證基本可用性
5. 通過後依規則晉升到 `test` / `pp`
6. 完成人工或自動審批後部署 `prod`
7. 發布完成後收集監控、健康檢查與部署報告

## 放行條件
1. 關鍵測試全部通過
2. 無阻斷性安全風險
3. 版本資訊、變更說明與 rollback plan 完整
4. 目標環境資源與相依服務正常

## 部署策略
| 策略 | 使用情境 | 說明 |
|---|---|---|
| Rolling | 一般穩定功能 | 逐步替換實例 |
| Blue/Green | 高穩定需求 | 新舊版本切換 |
| Canary | 高風險功能 | 小流量觀察後放大 |
| Hotfix | 緊急修復 | 優先處理關鍵問題 |

## 失敗與回滾
1. 若健康檢查失敗，立即停止 rollout
2. 系統自動切回上一個穩定版本
3. 建立 incident / bug ticket 並附上部署紀錄
4. 經分析後決定重新部署或轉 hotfix 流程

## 輸入與輸出
### 輸入
- 可部署版本
- 部署設定與環境變數
- 發版審批資訊

### 輸出
- 部署紀錄
- 版本對應環境狀態
- 回滾與變更報告

## 指標建議
1. 部署成功率
2. Deployment frequency
3. 回滾率
4. 變更失敗率
