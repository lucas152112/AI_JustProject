# Notification Workflows

版本：v0.5.1

## 流程目標
把系統事件統一轉換成可追蹤、可訂閱、可升級處理的通知，分發到站內與外部 IM 通道。

## 事件來源
1. 任務建立、指派、延遲
2. 測試失敗或部署完成
3. 專案里程碑變更
4. 監控告警與 DR 事件
5. 系統安全與權限異動

## 通知通道
| 通道 | 使用情境 |
|---|---|
| In-App | 站內通知中心 |
| Email | 正式通知、審批 |
| Telegram | 即時團隊提醒 |
| Discord | 開發協作與 bot 指令 |
| LINE / fastIM | 業務或外部協作場景 |

## 主流程
1. 業務模組產生事件並送入 notification service
2. 系統依事件類型、優先級與訂閱規則建立通知任務
3. 套用模板，產生標題、摘要、連結與動作按鈕
4. 依使用者偏好與權限選擇投遞通道
5. 發送後記錄 delivery status、read status 與 retry 結果
6. 重要通知可升級為 repeated reminder 或 escalated alert

## 優先級分級
| 等級 | 說明 |
|---|---|
| Critical | 服務中斷、安全事件、P0 問題 |
| High | 關鍵測試失敗、發版異常 |
| Medium | 任務異動、審核提醒 |
| Low | 一般資訊更新 |

## 例外處理
### 投遞失敗
1. 自動重試指定次數
2. 仍失敗則切換備援通道
3. 記錄錯誤並通知管理者

### 未讀超時
1. 關鍵通知超時未讀時升級提醒
2. 追加發送給主管、值班人員或備援群組

## 模板標準
1. 事件標題
2. 影響範圍
3. 處理連結
4. 發生時間
5. 責任人與狀態

## 指標建議
1. 送達率
2. 平均送達時間
3. 未讀逾時率
4. 關鍵告警確認時間
