# User Workflows

版本：v0.5.1

## 流程目標
整理一般使用者從登入、查詢、協作到回報問題的日常操作路徑，作為前台頁面與權限設計基礎。

## 主要角色
- 一般成員：查看專案、任務與測試結果
- PM / Lead：追蹤團隊進度與風險
- Reviewer / QA：處理驗收與測試相關資訊

## 核心使用流程
### 1. 登入與進入工作區
1. 使用者登入系統並完成 MFA
2. 系統載入個人首頁、待辦與最近專案
3. 顯示未讀通知、待審核項目與風險提醒

### 2. 查看與參與專案
1. 進入專案列表篩選我參與的專案
2. 查看專案概況、里程碑、版本與活動紀錄
3. 根據權限進行留言、更新狀態或查看成員分工

### 3. 任務處理
1. 在任務列表或看板中查看待辦
2. 開啟任務詳情，確認描述、附件、驗收條件
3. 更新進度、留言、附上 commit 或 PR 連結
4. 任務完成後標記為待審核或待測試

### 4. 查詢測試與部署結果
1. 使用者查看與自己任務相關的測試結果
2. 若版本已部署，可查詢當前環境狀態與變更內容
3. 發現問題時可直接建立 bug 或回報紀錄

## 例外場景
### 無權限
1. 顯示 403 與申請權限入口
2. 可送出權限申請給管理者

### 空資料
1. 若沒有專案或任務，顯示導引與建立入口
2. 提供常見操作說明與模板

### 需要支援
1. 透過通知中心、評論或回報機制發送問題
2. 關聯目前頁面、任務與版本上下文

## 關鍵頁面對應
1. `/`
2. `/projects`

4. `/tasks`
5. `/tasks/:taskId`
6. `/testing`
7. `/deployments`
8. `/notifications`

## 體驗指標
1. 登入成功率
2. 待辦處理完成率
3. 任務更新頻率
4. 通知讀取率
